start.sh - для установки виртуального окружения, подготовки данных и запуска скрипта

Для запуска в ручную (после установки виртуального окружения)
1) clear.sh - подготовка файлов
2) source venv/bin/activate (MAC)
3) ./main.py

Загрузка работает как по 1 комплекту файлов так и по нескольким.
В случае нескольких, происходит последовательная обработка.
При не совпадении дат или отсутствия полного комплекта файлов
выведется сообщение с названием недостающего файла.
Такая реализация необходима в для возможности произвести загрузку
за несколько дней (из-за нарушения поставки данных).

Данные terminals хранятся в SCD2 формате.

Дополнительно реализована обработка ошибок и логгирование.
